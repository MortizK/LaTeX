\section{Semantic Web}

Das wurde für Menschen gemacht, somit besteht es aus natürlicher Sprache welches mehrere Synonyme hat, wie aber auch Homonyme (gleiche Wörter mit unterschiedlichen Bedeutungen)

Zudem gibt es keine Bedeutung für die Verbindungen von Links, Bilder, usw. 

Das "The Semantic Web" (Breners-Lee, Hendler, Lassila; 2001) war der Grundbaustein. und wurde viel erweitert:

\begin{enumerate}
    \item WWW durch Semantik
    \item W3C-Standard
    \item Technik: zuerst RDF(S), später OWL

\begin{enumerate}
    \item Google Knowledge Graph (unterstützend für Websuchen)

    \item DBpedia (RDF-Format für Wikipedia)
    \item GraphRAG (für LLM)
\end{enumerate}

\subsection{Linked Data}

Linked Data is the Semantic Web done right - Berners-Lee, 2006

\begin{enumerate}
    \item Use URIS as names for things
    \item Use HTTP URIs so that people can look up those names.
    \item When someone looks up a URI, provide useful information, using standards (RDF*, SPARQL, OWL).
    \item Include links to other URIs so that they can discover more things (Link von Stuttgart zu Baden Württemberg)
\end{enumerate}

\subsection{RDF}

Resource Description Framework

Ein RDF-Graph ist Menge von Tripeln (die Ontologie)

\begin{enumerate}
    \item URI
    \item Literal: Zahl, Datum, String, Darstellungen
    \item Leere Knoten (blank node, Bnode): Für Existenz ohne sie zu benennen.
\end{enumerate}

Ein RDF-Tripel besteht aus Subjekt (Startknoten) - Prädikat (Kante) - Objekt (Zielknoten)

\begin{enumerate}
    \item Subjekt: URI oder Bnode
    \item Prädikat: URI
    \item Objekt: UIR oder Bnode oder Literal
\end{enumerate}

\subsubsection{Syntax für RDF: TTL}

Der Syntax wird in TTL/ "Turtle" geschrieben.

\begin{enumerate}
    \item URIs in spitzen Klammern
    \item Literale in Anführungszeichen
    \item Tripel durch Punkt abgeschlossen
    \item Leerzeichen und Zeilenumbrüche außerhalb von Bezeichnern sind irrelevant.
\end{enumerate}

\begin{lstlisting}
<http://example.org/SemanticWeb> <http://example.org/verlegtBei>
    <http://springer.com/Verlag> .
<http://example.org/SemanticWeb> <http://example.org/titel>
    "Semantic Web - Grundlagen" .
<http://springer.com/Verlag> <http://example.org/name> "Springer Verlag" .
\end{lstlisting}

Wir können auch abkürzen

\begin{lstlisting}
@prefix ex: <http://example.org/> .
@prefix springer: <http://springer.com/> .
@prefix : <http://jan.org/ontology/> .

ex:SemanticWeb ex:verlegtBei springer:Verlag .
ex:SemanticWeb ex:titel "Semantic Web - Grundlagen" .
springer:Verlag ex:name "Springer Verlag" .
:jan :vorname "Jan".
\end{lstlisting}

Jetzt gab es eine Übung S.83 Diese sollten wir Zeichnen und als TTL aufschreiben.

Ich benutze hier die Abkürzung der Abkürzung. So kann in Zeile 4 das ex:JanHladik weggelassen werden.

\begin{lstlisting}
@prefix dhbw: <http://dhbw.de/> .
@prefix ex: <http://example.org/> .

ex:JanHladik    ex:name     "Jan Hladik" ;
                ex:haelt    dhbw:SW .
dhbw:SW         ex:name     "Semantic Web" . 
\end{lstlisting}

Pitfalls:

\begin{enumerate}
    \item Das ; vergessen (übernimmt das Subjekt in die nächste Zeile)
    \item Das / am Ende vom Präfix vergessen
    \item Das . nach dem Präfix vergessen
    \item Eigentlich " statt ' % Muss mal gucken wie ich das im Compiler fixe
\end{enumerate}

\marginpar{Hausaufgabe S.105}

Entscheiden Sie, ob die folgenden Aussagen wahr oder falsch sind:

\begin{enumerate}
    \item URIs können für beliebige Satzteile (Subjekt/Prädikat/Objekt) eines Tripels stehen.


    \item Leere Knoten können für beliebige Satzteile stehen.


    \item Literale können für beliebige Satzteile stehen.


    \item Zwei leere Knoten mit unterschiedlicher ID können für die gleiche Ressource stehen.


    \item Zwei unterschiedliche URIs können für die gleiche Ressource stehen.


    \item Kommen in mehreren RDF-Dokumenten leere Knoten mit der selben ID vor, dann müssen
    sie für dieselbe Ressource stehen.


    \item Kommen in mehreren RDF-Dokumenten die gleichen URIs vor, dann müssen sie für
    dieselbe Ressource stehen.


    \item Zwei unterschiedliche Literale können für den gleichen Wert stehen.


    \item Zwei Literale unterschiedlichen Datentyps können für den gleichen Wert stehen.


\end{enumerate}
