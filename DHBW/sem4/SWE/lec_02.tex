\section{Software-Metriken}

% Holen wir noch aus dem Sem3 nach.

Ein wichtiger Begriff um Vorschritt oder Leistung zu messen ist KPI (Key Performance Indikator). In der Software Entwicklung heißt das Metriken.

\subsection{Definition}

A quantitative measure of the degree to which a system, component, or process possesses a given attribute.

Ziele:

\begin{enumerate}
    \item Bewertung der Qualität des Produktes oder Prozesses
    \item Prognosen
    \item Unterstützung von Entscheidungen oder Begründungen.
\end{enumerate}

Ich kann anhand von Kennzahlen, Begründet und Objektiv Entscheidungen treffen.

\begin{enumerate}
    \item Kosten - Kostenmetrik
    \item Anzahl Fehler - Fehlermetrik
    \item Volumen und Umfang - Volumen-, Umfangsmetrik
    \item Qualität - Qualitätsmetrik
\end{enumerate}

\subsection{Eigenschaften}

Eine Metrik muss die folgenden Eigenschaften aufweisen:

\begin{enumerate}
    \item differenziert
    \item vergleichbar
    \item reproduzierbar
    \item verfügbar
    \item relevanten
    \item rentabel
    \item plausibel
\end{enumerate}

Neben diesen gibt es auch noch SMART-Attribute. Diese sind Ähnlich wie die oben.

\subsection{Arten}

Es wird zwischen drei Metriken unterschieden:

\begin{enumerate}
    \item objektive Metrik
    \item subjektive Metrik
    \item Pseudometriken
\end{enumerate}

\subsection{Komplexitätsmaß nach McCabe}

Der hat für Code eine Objektive Metrik entwickelt. Hierzu gibt es die zyklomatische Zahl aus der Graphentheorie. Es geht darum, die Komplexität eines systems zu reduzieren.

\subsection{Kopplung und Zusammenhalt}

\textbf{coupling}: The manner and degree of interdependence between software modules.

\textbf{cohesion}: The manner and degree to which tasks performed by a single software module are related to one another.

\subsection{Identifikation und Entwicklung von Metriken}

Ein Ansatz ist der Goal-Question-Metric-Ansatz:

\begin{enumerate}
    \item Definiere relevante Ziele
    \item Leite aus jedem Ziel Fragen ab, die beantwortet werden müssen, um zu prüfen, od bas Zeil erreicht wurde.
    \item Lege für jede Frage Metriken fest, die bei der Beantwortung der Frage helfen.
\end{enumerate}
