\section{Switching \& Routing}

\subsection{Switching}

\begin{enumerate}
    \item Cut-Through Bridge
    
    Paket wird bis zur Zieladresse eingelesen und dann bereits ausgesendet $\rightarrow$ minimale Verzögerung 
    \item Store-and-Forward Bridge
    
    Paket wird komplett eingelesen, auf Fehler geprüft und dann weitergelesen $\rightarrow$ Verzögerung abhängig von Paketgröße
    \item Link-Aggregation ("Trunking")
    
    Gruppierung von physikalischen Ports zu einer logischen Port Verbindung. Standard 802.3 mit Load-balancing
\end{enumerate}

\subsubsection{MAC-Adresse}

Für einen Broadcast werden alle Adressbits auf 1 gesetzt.

Für einen Multicast werden die niedrigsten Bits im erstem Oktett gesetzt. z.B. 01:80... oder 03:80...

\subsubsection{Flusskontrolle - Flow Control}

Kann kein weiteres Paket mehr akzeptiert werden, da Puffer voll ist. Die resultiert in ein "Wandering Backpressure", was alle Switches Stück für Stück blockiert.

Funktioniert bis einer ein längeres Problem hat. Kann im schlimmsten fall den Backbone lahmlegen.

Eingrenzung von Broadcast können bei Switches so implementiert werden:

\begin{enumerate}
    \item Broadcast-Limiter (pro Sekunde)
    \item VLANs
    \item Filter (z.B. bei bestimmten Sendern/ Empfänger blockieren/ erlauben)
    \item Router (schickt keinen Broadcast weiter)
\end{enumerate}

\subsection{VLANs}

\subsection{Spanning Tree}

Zu Spanning Tree kommt eine \textbf{Klausuraufgabe} mit einigen Punkten (Für eine Ganze Note)

Jede Brücke ist eindeutig:

\begin{enumerate}
    \item ID (8 oktett): Prio (4 bit) + Inst. (12 bit) + Mac-Adr. (6 oktett)
    \item Root Pfadkosten (Kosten von Brücke zur Root)
    \item Port ID (2 okt.): Port-Prio (4 bits) + Port-Nr (niedr. 12 Bits) Port mit kleinster Port-ID besitzt höchste Priorität
\end{enumerate}

Der Algorithmus:

\begin{enumerate}
    \item Switch mit kleinster Brücken-ID wird Root-Bridge
    \item Port mit geringsten Root-Pfadkosten wird Root-Port. Bei gleichen Pfadkosten entscheidet die Port-Prio.
    \item Bei LAN (Hub) entscheidt die BID der Brücken
\end{enumerate}

\subsection{Router}

% Startet auf Folie 6-37

Synonym: Gateway

Einordnung: Schicht 3 des OSI-Referenzmodells

Netzzugriff: Benötigt pro Port eine logische Adresse (z.B. IP-Adresse)

Alle Daten (z.B. Emails) werden mehrfach gekapselt und während der Reise teilweise ausgepackt und wieder erneut verpackt, bis der Finale Adressat erreicht ist und dieser die Nachricht auspacken kann.

Das geroutete Netz auf Folie 6-44 ist so aufgebaut, das zwischen Router 1 und Router 2 das Netz 10.12.0.0/24 ist (zu lesen 10.Eins zu Zwei.0.0/24) In so einem Netz müssen wir den Router neben der Default-Route noch explizierte Routen geben (Routing-Tabelle), die nicht über die Default-Route gefunden werden können. Je präziser eine Route (das Netz oder /n), je höher ist die Priorität in der Tabelle.

Die /24 steht für die Netzmaske 255.255.255.0 somit gehören die ersten 24 bit zu dem Netz.
