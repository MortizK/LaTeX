\subsection{k-Means}

Wir wählen den Parameter $k$

\[
argmin_S \sum_{j=1}^{k}\sum_{x^[(i)]\in S^{(i)}} \left\|x^{(i)-\mu^{(i)}}\right\|^2
\]

Schritte:

\begin{enumerate}
    \item $k$ wählen
    \item Zufällige Mittelpunkte initialisieren
    \item Zuordnung der Punkte zu den Mittelpunkten (Voronoi regions)
    \item Update die Mittelpunkte
    \item Bis zur convergence durchführen
\end{enumerate}

Kann in einem lokalem minimum landen. Deshalb wird dies häufig mehrfach ausgeführt auch mit unterschiedlichen $k$s

k-means geht davon aus das die Cluster rund sind und gleich groß. Bei unterschiedlich großen Cluster wird in der Mitte geteilt. Ist eine Hart Zuordnung (crisp)

\subsubsection*{Number of Clusters}

Bayesian Information Criterion (BIC). Ist ein Wert um ein k für das Model zu schätzen.

Es legt ins Verhältnis wie komplex das model ist und wie gut das model ist.

Occam's razor: prefer the simplest model among equally good ones

\subsection{Gaussian Mixture Models (GMM)}

Die Idee ist es, mehrere Normalverteilungen über das Datenset zu legen. Dies erlaubt mehrere elliptische Cluster.

Das Model wird trainiert mit dem Expectation Maximization (EM) und wird bis zur Konvergenz Trainiert.

\begin{enumerate}
    \item Die Ursprünglichen Verteilungen werden zufällig gewählt.
    \item Schritt E:
    
    Wir gehen davon aus, dass wir die Verteilungen kennen und berechnen für jeden Punkt die Wahrscheinlichkeit, das diese zu dem jeweiligen Cluster gehört.
    \item Schritte M:
    
    Nun haben wir die Wahrscheinlichkeiten und berechnen daraus die neuen Verteilungen.
    \item E und M wiederholen bis zu Konvergenz.
\end{enumerate}

\subsection{Hierarchical Clustering}

Visualisierung in einem "dendrogram" (Sieht aus wie ein Binärbaum). Es gibt zwei Varianten:

\begin{enumerate}
    \item Agglomerative/ Bottom up
    \item Divisive/ top down
\end{enumerate}

Wie funktionieren die Merges von zwei Zweigen? Wir können die Cluster anhand ihrer Distanz mergen:

\begin{enumerate}
    \item Average linkage
    \item Complete linkage (max und globales Kriterium)
    \item Single linkage (min und Lokales Kriterium)
\end{enumerate}

Für min und max müssen wir NxN Entfernungen ausrechnen und speichern. Als Lookup für die nächsten merges wird dann immer das minimum von einem der beiden Linkages genommen.

Single Linkages resultiert tendenziell in langen Clusters, die in keinem balancierte Baum resultiert.

Complete Linkage kann häufiger in einem balancierten Baum.

Frage: Wo wird die trennlinie gezogen um die Anzahl der Cluster zu bestimmen? Evtl. Wo lange die Anzahl der Cluster sich nicht ändert (bezogen auf die Entfernung, wo ein Merge passiert.)

\subsection{DBSCAN}

Density-Based Spatial Clustering of Application with Noise

Zwei Kriterien:

\begin{enumerate}
    \item Nicht weiter weg als $\epsilon$
    \item Es muss eine gewisse Anzahl an Punkten (minPts) im Bereich von dem anderem Punkt sein.
\end{enumerate}

In einem Cluster sind alle Punkte untereinander dichte-erreichbar voneinander.

Es wird unterschieden in:

\begin{enumerate}
    \item Core Points
    \item Density-reachable Points (Können von einem Core Point erreicht werden)
    \item Noise points
\end{enumerate}

Vorteile: Wir haben beliebig viele Cluster, ein Verständnis von Rauschen, 

Nachteile: Ist abhängig von der Distanccfunktion und bei Daten mit unterschiedlichen Dichten kann $\epsilon$ schwer gewählt werden.

\subsection{Mean Shift Segmentation}

Wird häufig für Bilder verwendet. z.B. Kantenerkennung.

Funktioniert über gradient Descent (Vorher Dichte Schätzung). Alles was im gleichem Maximum ended ist im gleichem Cluster.

Die Dichte Funktion wird erstellt durch die Aufsummierung von Normalverteilungen um jeden einzelnen Punkt.

Der Gradient Descent wird über ein Moving Mean umgesetzt. So werden gewichtete Mittelwerte erstellt, statt die tatsächliche Dichte auszurechnen.

Ein gutes $h$ für die breite der Gausfunktion kann nach Scott'r Regel ausprobiert werden:

\[
h = N^{-\frac{1}{p+4}}
\]

mit $p$ die Anzahl der Dimensionen  und $N$ die Anzahl der Datenpunkte.

\subsection{Beurteilung von Clustering}

Wenn wir Punkte habe, wo wir das Cluster kennen. Können wir überprüfen ob alle der Punkte die im gleichem Cluster sein sollen auch im gleichem sind.

Daraus lässt sich die Purity berechnen. (=1 Wenn jeder Punkt sein eigenes Cluster ist)

Ohne Daten-Labels gibt es auch:

\begin{enumerate}
    \item Cluster cohesion: Wie eng sind Punkte in einem Cluster
    \item Cluster separation: Wie unterschiedlich sind mehrere Cluster
\end{enumerate}

Bekannt ist der Davies-Boulding Index

\[
DB = \frac{1}{k} \sum_{i=1}^{k} max_{i\neq j} \left(\frac{d_j + d_j}{d_{ij}}\right)
\]

oder der Silhouett Score.
